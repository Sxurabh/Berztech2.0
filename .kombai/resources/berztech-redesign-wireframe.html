REDESIGN CONCEPT 2: Accessible Contact & Auth Forms

Redesign 2: Accessible Form Design

✓ Key Improvements:
  • Visible focus indicators on all inputs (3px blue outline)
  • Required field indicators with asterisk AND aria-required
  • Error state with icon + message (not color-only)
  • Labels properly associated via id/htmlFor (not placeholder-only)
  • Success/Loading states with aria-live announcements
  • Min 44x44px touch targets on all buttons

IMPLEMENTATION ROADMAP

Redesign Implementation Guide

Phase 1: Performance (Week 1-2)

  • ✓ Remove animation from initial hero render (use loading state instead)
  • ✓ Refactor AnimatedNumber to use requestAnimationFrame + CSS
  • ✓ Convert ProcessStrip animations to CSS transitions (remove Framer Motion)
  • ✓ Add prefers-reduced-motion support globally
  • ✓ Measure LCP/CLS/INP with Chrome DevTools (target: <2.5s / <0.1 / <100ms)

Phase 2: Accessibility (Week 2-3)

  • ✓ Add focus indicators to all interactive elements
  • ✓ Fix color contrast issues (scrollbar, links, text)
  • ✓ Add proper ARIA labels, roles, and live regions
  • ✓ Implement visible required field indicators
  • ✓ Add error/success messaging with icons
  • ✓ Test with keyboard navigation and screen reader

Phase 3: Mobile & UX (Week 3-4)

  • ✓ Increase touch targets to 44x44px minimum
  • ✓ Fix dashboard layout (remove horizontal scroll)
  • ✓ Make request modal full-screen on mobile
  • ✓ Add loading/error states to forms and auth
  • ✓ Test on actual devices (iOS Safari, Android Chrome)

Phase 4: Validation & Deployment (Week 4)

  • ✓ Run Lighthouse audit (target: 90+ Performance, 100 Accessibility)
  • ✓ Run axe DevTools accessibility check
  • ✓ User testing with keyboard-only navigation
  • ✓ Monitor Core Web Vitals in production
  • ✓ Set up Lighthouse CI for future deployments

Quick Wins (Can implement immediately)

  • Add focus rings: Add to globals.css: *:focus-visible { outline: 3px solid #3b82f6; outline-offset: 2px; }
  • Improve scrollbar contrast: Change rgba(0,0,0,0.1) → rgba(0,0,0,0.3) in globals.css
  • Add skip link: Add hidden "Skip to content" link at top of Header component
  • Increase button sizes: Change service card icons from 40px to 48px in Services section
  • Add prefers-reduced-motion: Wrap animation code in media query check
